# 产品范围

## 1. 目标用户

- 需要在消费级 GPU 上训练或微调开源 LLM 的算法工程师。
- 需要复现实验、比较模型和管理 Checkpoint 的研究人员。
- 需要展示分布式训练与 LLMOps 能力的个人开发者。
- 不具备 A100/H100 集群，但拥有多张 RTX 3090/V100 的实验室。

## 2. 核心用户故事

### 训练工程师

> 我希望提交一个模型、数据和资源配置，系统自动检查硬件、推荐并行策略，并生成可恢复的训练任务。

### 研究人员

> 我希望从一个 Run ID 还原数据、代码、配置、环境和模型结果，以便验证实验是否可复现。

### 模型负责人

> 我希望候选模型只有在通过能力和性能门禁后才能进入部署阶段。

### 推理工程师

> 我希望将晋级后的模型一条命令部署，并获得统一的延迟、吞吐和显存报告。

## 3. 产品边界

TinyLLM-System 管理的是单机多 GPU LLM 生命周期。

MVP 不覆盖：

- 多节点训练。
- 大规模调度集群。
- Kubernetes 多租户平台。
- 商业计费。
- 百亿级从零预训练。
- 自研 CUDA 算子。
- 完整 RLHF 平台。

## 4. 成功标准

- 能在共享的 10×RTX 3090 主机上稳定完成可复现的 1/2/4 卡实验，并如实记录资源边界。
- 能训练单卡放不下的模型。
- 能自动恢复分布式训练。
- 能自动生成基线与候选模型对比。
- 能阻止回退模型晋级。
- 能部署并压测生产候选模型。
- 能根据 Run ID 还原实验上下文。
